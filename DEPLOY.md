# 🚀 הוראות להעלאת NoExcuse לשרת חיצוני

האפליקציה עכשיו מוכנה להפצה (Production Ready)! איחדתי את הממשק והשרת כך שתוכל להריץ אותם כיחידה אחת בכל שירות ענן.

### מה השתנה?
1. **API יחסי:** האפליקציה כבר לא מחפשת רק את `localhost`, אלא יודעת לפנות לשרת שבו היא נמצאת.
2. **איחוד:** השרת (Backend) עכשיו מגיש בעצמו את הממשק (Frontend) מתוך תיקיית `client/dist`.
3. **PWA:** האפליקציה מוכנה להתקנה בנייד ברגע שתעלה לאוויר.

### 🛠️ איך מעלים? (אפשרויות מומלצות)

#### אופציה א': Railway / Render (הכי קל)
1. צור חשבון ב-[Railway.app](https://railway.app) או [Render.com](https://render.com).
2. חבר את חשבון ה-GitHub שלך.
3. בחר את ה-Repository של הפרויקט.
4. השירות יזהה אוטומטית את ה-`package.json` ויציע להריץ את `npm start`.
5. **חשוב:** וודא שביצעת `npm run build` בתוך תיקיית ה-`client` לפני שאתה מעלה (כבר עשיתי את זה עבורך כרגע!).

#### אופציה ב': VPS פרטי (שרת לינוקס)
1. התקן Node.js על השרת.
2. העתק את כל הקבצים.
3. הרץ פעם אחת `npm install` בתיקייה הראשית.
4. השתמש ב-PM2 כדי להשאיר את האפליקציה בחיים:
   ```bash
   pm2 start server.js --name noexcuse
   ```

### 📝 הערות חשובות:
* **שמירת נתונים:** כרגע הנתונים נשמרים בקובץ `db.json`. בשרתים כמו Vercel שמוחקים קבצים בכל פריסה, הנתונים יאופסו. בשרתים כמו Railway או VPS הם יישמרו.
* **HTTPS:** כדי שהתקנה בנייד (PWA) תעבוד, השרת חייב להיות עם תעודת SSL (רוב שירותי הענן נותנים את זה אוטומטית).
